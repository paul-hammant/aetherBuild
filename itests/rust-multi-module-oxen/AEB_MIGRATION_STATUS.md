# Cargo workspace to aetherBuild Migration Status

Upstream: https://github.com/Oxen-AI/Oxen.git

## Status: aeb owns the Cargo.toml

aeb now generates the workspace `Cargo.toml` and all member `Cargo.toml` files
from `.build.ae` DSL. Cargo is only invoked for dep resolution, source
compilation, and linking.

## Modules

| Module | Compile | Tests | Notes |
|--------|---------|-------|-------|
| crates/lib (liboxen) | OK (110+ deps) | — | Compiles from source via cargo check |
| crates/cli (oxen-cli) | OK | — | |
| crates/server (oxen-server) | compiles to .o, link fails | — | `-lduckdb` system library missing on NUC |
| crates/oxen-py | — | — | Skipped (needs PyO3 + Python) |

The link failure is environmental: `libduckdb-dev` is not installed on this NUC.
Running `cargo build` directly fails identically. aeb's workspace generation is
correct — cargo resolves 130+ deps, downloads crates, and compiles from source
through the entire workspace before hitting the missing system library.

## What aeb owns

- **Root `Cargo.toml`** — generated by `rust.cargo_workspace(b)`:
  - `[workspace]` with `resolver`, `members`, `exclude`
  - `[workspace.package]` with version, edition, license-file, description,
    homepage, repository, readme, rust-version, keywords, categories
  - `[workspace.dependencies]` — all 132 deps loaded from `workspace.deps.ae`
- **Member `Cargo.toml` files** — generated by `rust.cargo_crate(b)`:
  - `[package]` with `name`, `edition.workspace = true`
  - `[[bin]]` for oxen-cli and oxen-server
  - `[dependencies]` with `foo.workspace = true` refs
  - `[dev-dependencies]` for test-only deps (actix-multipart-test)
  - `[features]` for feature flags (oxen-server's `otel`)

## What cargo still does

- Dep resolution and download from crates.io
- Source compilation (rustc invocation, proc macros, build scripts)
- Linking and binary output
- Feature resolution across workspace members

## New Rust SDK features (since last update)

### Workspace generation
- `rust.cargo_workspace(b)` — generates workspace root Cargo.toml
- `rust.cargo_crate(b)` — generates member Cargo.toml
- `workspace_member`, `workspace_version`, `workspace_edition`, `workspace_resolver`
- `workspace_license_file`, `workspace_description`, `workspace_homepage`,
  `workspace_repository`, `workspace_readme`, `workspace_rust_version`,
  `workspace_keywords`, `workspace_categories`, `workspace_exclude`
- `workspace_dep(name, ver)` — simple workspace dep
- `workspace_dep_features(name, ver, feats_csv)` — dep with features
- `workspace_dep_full(name, ver, feats, pkg, nodef)` — dep with package alias
  and `default-features = false` support
- `load_workspace_deps(path)` — load deps from a `workspace.deps.ae` file

### Member-level
- `ws_dep(name)` — reference a workspace dep in `[dependencies]`
- `dev_ws_dep(name)` — reference a workspace dep in `[dev-dependencies]`
- `path_dep(name, path)` — local path dep (e.g., `liboxen = { path = "../lib" }`)
- `feature(name, deps_csv)` — declare a feature flag
- `default_features(csv)` — set the `default = [...]` list
- `bin(name, path)` — declare `[[bin]]` entry
- `edition`, `version`, `crate_type`, `lib_path` — inherited/package fields

### Converter tools
- `tools/cargo-to-deps-ae.sh` — converts upstream Cargo.toml
  `[workspace.dependencies]` to `workspace.deps.ae` format. Detects
  `default-features = false` and `package = "alias"` and emits
  `workspace_dep_full()` accordingly.
- `tools/cargo-member-to-ae.sh` — converts member Cargo.toml to a
  starter `.build.ae` file.

## Known issues

1. **`-lduckdb` link error** — system `libduckdb-dev` not installed on NUC.
   Not an aeb issue. Install `libduckdb-dev` or use docker to fix.
2. **`oxen-py` skipped** — PyO3 build needs Python headers and maturin.
3. **`[package.metadata.*]` passthrough** — metadata blocks like `docs.rs`
   and `cargo-machete` are not preserved in generated member Cargo.toml.
   Not blocking compilation.
